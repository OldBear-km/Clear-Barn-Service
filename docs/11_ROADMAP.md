# Roadmap — План развития системы Clear Barn

## 1. Принцип развития

Развитие системы строится по этапам:
- сначала стабильное ядро
- затем стабилизация
- затем масштабирование
- затем расширение функционала

Архитектурная целостность важнее скорости.

—

## ЭТАП 1 — Ядро системы (MVP)

Цель: стабильная и безопасная основа.

### 1.1 Инфраструктура
- Docker Compose
- FRP (FRPS + FRPC)
- Nginx на VPS
- TLS сертификаты
- Настроенный webhook Telegram

### 1.2 Аутентификация
- JWT
- Telegram Mini App проверка подписи
- RBAC
- Tenant isolation

### 1.3 Основные модули
- Users
- Organizations
- Employments
- Shifts
- Payroll
- Bonuses/Penalties

### 1.4 Фоновые процессы
- автозакрытие смен
- напоминания
- логирование

### 1.5 Audit
- полная история изменений

Результат:
Система готова к использованию в 1–2 филиалах.

—

## ЭТАП 2 — Стабилизация

Цель: сделать систему надежной.
- unit тесты ключевых сервисов
- интеграционные тесты payroll flow
- улучшение логирования
- улучшение обработки ошибок
- rate limiting
- улучшение UX Mini App

—

## ЭТАП 3 — Масштабирование

Цель: подготовка к росту сети.
- Celery worker
- разделение очередей уведомлений
- оптимизация индексов
- мониторинг (Sentry/Grafana)
- разделение VPS и backend серверов

—

## ЭТАП 4 — Расширение функционала

Возможные направления:
- управление выручкой
- учёт расходов
- инвентаризация
- печать этикеток (полная интеграция)
- расширенные отчеты
- глобальная аналитика по сети

—

## ЭТАП 5 — Enterprise

- разделение сервисов (microservices при необходимости)
- горизонтальное масштабирование
- multi-region
- SLA мониторинг
- permission-based модель

—

## Приоритеты

1. Безопасность
2. Стабильность
3. Простота
4. Масштабируемость
5. Новый функционал

—

## Чего НЕ делать

- не добавлять новые модули до стабилизации payroll
- не переходить на Kubernetes раньше времени
- не усложнять архитектуру без необходимости
- не нарушать tenant isolation ради скорости

—
